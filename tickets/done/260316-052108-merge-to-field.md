# チケット単位で merge 先ブランチを指定できるようにする

## 概要

現在 `close` 時のマージ先は `.ticket-config.yaml` の `default_branch` のみ。
チケットの YAML フロントマターに `merge_to` フィールドを追加し、チケット単位でマージ先を上書きできるようにする。

**ユースケース:** 複数チケットを中間ブランチにまとめてから main にマージしたい場合。

## 設計

- チケット YAML に `merge_to` フィールドを追加（デフォルト値: "default"）
- `new` コマンドのデフォルトテンプレートに `merge_to: default` を含める
- `close` 時: `merge_to` が設定されていればそちらを優先、なければ従来通り `default_branch`
- `merge_to` 先ブランチが存在しない場合はエラー
- 空、null、"default"（大文字小文字不問）はすべて config の `default_branch` にフォールバック
- `auto_push`, `delete_remote_on_close` の挙動は変更なし
